## 1. v0.7.0 Goal

v0.5.x/v0.6.0 added many features without integration tests. v0.7.0 fills the gap:

- web-ext + headless Firefox integration test framework
- Core action end-to-end tests (navigate/click/fill/screenshot)

## 2. Core Design

### 2.1 Integration Test Architecture

```
Integration Test Runner (Node.js)
    │
    ├── 1. Start Bridge (standalone mode, fixed port)
    ├── 2. Start headless Firefox with web-ext
    ├── 3. Firefox loads BRP extension
    ├── 4. Extension connects to Bridge WS
    ├── 5. Test runner sends JSON-RPC → Bridge → Extension → Firefox
    ├── 6. Assert Firefox state (tabs/DOM/screenshots)
    └── 7. Cleanup
```

### 2.2 Test Categories

| Category | What | How |
|---|---|---|
| Contract | Bridge ↔ Extension message format | Mock WS, verify JSON-RPC shape |
| E2E Action | navigate/click/fill/screenshot | web-ext + headless Firefox |
| E2E Permission | E1 dialog / E2 blacklist / Allowlist | web-ext + headless Firefox |
| E2E Tab | tab.open/close/setControllable | web-ext + headless Firefox |

### 2.3 CI Integration

- Integration tests run on PR + main push
- ubuntu-only (Firefox headless)
- Separate job from unit tests (slower)
- 10min timeout (Firefox startup overhead)

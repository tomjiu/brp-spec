# v0.4.2 Implementation Plan

> Integration Testing + Conditional MRU

**Status**: Draft
**Prerequisite**: v0.4.1 released (tag `v0.4.1`)
**Companion**: [`docs/v0.4.1-IMPLEMENTATION-PLAN.md`](v0.4.1-IMPLEMENTATION-PLAN.md)

---

## 1. v0.4.2 Goal

v0.4.0 (B1) + v0.4.1 (E3/E4) made the call chain complex:
```
AI → Bridge → Background → Content Script → DOM
```

Unit tests can't cover integration paths. Race conditions found in B1 (port.disconnect vs stdin EOF, token not stored) were all integration issues. v0.4.2 establishes integration test baseline.

### Non-goals (deferred)

| Item | Deferred to | Reason |
|------|:---:|--------|
| Defense in Depth (permission gating) | v0.5.0 | Stability first |
| supportsAutoLink capability | v1.0-pre | Not needed until multiple versions |

## 2. Sub-PR Breakdown (6-7 PRs)

### PR #32: Phase 1 — Smoke Test (2-3 days)

**Goal**: Verify B1 auto-link full chain in headless Firefox.

**Tool**: Playwright (modern API, good Firefox support)

**Scope**:
- New `tests/integration/` directory
- `tests/integration/package.json` (playwright + firefox)
- `tests/integration/fixtures/` (test HTML pages)
- `tests/integration/smoke.test.ts`:
  - Startup: headless Firefox + load extension
  - Test: bridge auto-starts via connectNative
  - Test: WS connection established
  - Test: initialize handshake succeeds
- CI: new integration job (ubuntu-only, PR-triggered)

**Acceptance**:
- Headless Firefox loads extension
- Bridge auto-starts (connectNative works)
- WS + initialize handshake succeeds
- CI ubuntu integration job green

**Risk**: web-ext/geckodriver version compat; headless Firefox native messaging support

---

### PR #33: Phase 2 — Core Operations (2-3 days)

**Goal**: Cover navigate/click/fill/type + E3 precondition + E4 fallback.

**Scope**:
- Test fixtures: `login-page.html`, `spa.html` (dynamic class for E4)
- Core operation tests:
  - navigate to URL
  - click element
  - fill input
  - type text
  - get interaction tree
- E3 Precondition tests:
  - reject click when tagName mismatches → `BRP_PRECONDITION_FAILED`
  - execute click when precondition matches
- E4 Fallback tests:
  - CSS selector fails → fallback to text → `matchedSelector: {type: "text"}`

**Acceptance**:
- All core operation tests pass
- E3 reject path verified
- E4 fallback chain verified

---

### PR #34: Phase 3 — Boundary Cases (1-2 days)

**Goal**: Cover extension unload, bridge crash, WS disconnect, multi-tab.

**Scope**:
- Extension unload → bridge exits (stdin EOF)
- Bridge crash → extension detects, reconnects
- WS disconnect → reconnect logic
- Multi-tab: shared background, single WS, per-tab operations

**Acceptance**:
- Unload/crash/reconnect scenarios pass
- Multi-tab behavior correct

---

### PR #35: (Conditional) MRU Multi-Instance Reuse (2-3 days)

**Prerequisite**: Phase 1-3 complete. Measure reconnect time.

| Reconnect time | Decision |
|---|---|
| < 1s | MRU low value, skip |
| > 3s | MRU worth doing |

**Scope (if done)**:
- `ipc_unix.rs` + `ipc_windows.rs`: MRU selection logic
- `lockfile.rs`: multi-instance registration
- `native.ts`: query available bridge before starting new one
- New `supportsMRU` capability

**Acceptance**:
- Reconnect time drops from Xs to < 500ms
- Multiple extension instances share one bridge

## 3. Tool Decision

**Playwright** chosen for integration testing:

- Modern API, clean
- Good Firefox headless support
- Active community
- Alternative: web-ext + selenium (heavier), raw `firefox --headless` (lighter but more manual)

---

## 4. Risk Register

| Risk | Severity | Mitigation |
|------|:---:|------------|
| Headless Firefox native messaging support | High | Phase 1 smoke test validates early; fallback to non-headless if needed |
| CI Firefox install/version drift | Medium | Pin Firefox version in CI |
| Integration test flaky on CI | Medium | Retry mechanism, generous timeouts |
| Playwright Firefox extension loading | Medium | Use `--load-extension` flag or temporary profile |
| MRU adds complexity without clear benefit | Low | Conditional on Phase 1-3 baseline data |

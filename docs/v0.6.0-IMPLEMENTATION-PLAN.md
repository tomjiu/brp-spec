## 2. Core Design

### 2.1 ts-rs Type Alignment

- Bridge: ts-rs already generates types to `bridge/bindings/` (v0.4.0-alpha)
- Extension: currently hand-writes types in `src/types.ts` / `src/types.content.ts`
- Change: build script copies `bridge/bindings/*.ts` → `extension/src/generated/`
- Extension imports from `./generated/` instead of hand-written
- `.gitignore` ignores `extension/src/generated/` (build artifact)
- CI verifies: `cargo test` generates types, `npm run build` copies + typecheck

### 2.2 tabGroups Coloring (Experimental)

- manifest adds `"tabGroups"` permission (Firefox v139+)
- AI `tab.open` / `page.navigate` → auto-add tab to "BRP" group (blue)
- AI request in progress → group color green
- Request failed (BRP_PERMISSION_DENIED) → group color yellow → remove from group
- `browser.tabGroups` not available → silent fallback to page-indicator (v0.5.1)

### 2.3 Fallback Strategy

```typescript
const tabGroupsSupported = typeof browser.tabGroups !== "undefined";

if (tabGroupsSupported) {
  // use tabGroups coloring
} else {
  // fall back to page-indicator (v0.5.1)
}
```

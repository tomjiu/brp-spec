# v0.5.0 Implementation Plan

> Defense in Depth — Permission Gating + Domain Blacklist + Security Hardening

**Status**: Draft
**Prerequisite**: v0.4.1 released (tag `v0.4.1`)

---

## 1. v0.5.0 Goal

v0.4.0 (B1) + v0.4.1 (E3/E4) gave AI basic browser control. v0.5.0 adds safety controls:

- **E1 Permission gating** — confirm before risky operations (navigation, form submit, script.execute)
- **E2 Domain blacklist** — user-configurable list of blocked domains
- **E5 Screenshot + log hardening** — permission gate for screenshots, log sanitization
- **B2 Multi-token** — per-client tokens with independent revocation

### Non-goals (deferred)

| Item | Deferred to | Reason |
|------|:---:|--------|
| Integration tests | v1.0 candidate | Code still evolving |
| Multi-instance MRU | v1.0-pre | Not needed until stable API |
| supportsAutoLink capability | v1.0-pre | Not needed until multiple bridge versions |
